## Watermarking for Licensed Distribution

Binary outputs generated by this repository for commercial distribution **MUST** be watermarked using the integrated watermarking system. This ensures:

1. **Traceability**: Each binary contains unique licensing information
2. **Authorization**: Only authorized parties can create valid watermarked binaries
3. **Compliance**: Watermarking enforces adherence to license terms
4. **Verification**: Licensing status can be verified cryptographically

### Requirements for Commercial Distribution

All commercially distributed binaries must:
- Contain a valid cryptographic watermark with unique License ID
- Include accurate User/Organization information
- Be created using the official watermarking tools provided in this repository
- Maintain the integrity of the watermark throughout distribution

### Watermarking Tools

Licensed users are provided with:
- Watermark creation scripts (`scripts/create_watermarked_binary.py`)
- Watermark verification tools (`scripts/verify_watermark.py`)
- Unique License ID and secret key for watermark signing
- Complete documentation (`docs/WATERMARK_DOCUMENTATION.md`)

### Prohibited Activities

The following activities are **strictly prohibited**:
- Distribution of non-watermarked binaries for commercial purposes
- Removal or tampering with watermarks from licensed binaries
- Sharing of watermarking secrets or License IDs
- Creation of watermarked binaries without proper authorization
- Unauthorized commercial data teleportation or redistribution

### Verification

Any party receiving a commercially distributed binary may verify its licensing status using the verification tools, provided they have access to the appropriate secret key.
